# 互動、刷新回饋、StoryScript 與圖示層規格

日期：2026-07-13  
範圍：CodexUsageMonitor 原生 Win32／Direct2D 小工具

## 1. 目標與已確認問題

本次修正針對目前可觀察到的四組問題：

1. 完整模式與簡單模式的分層懸浮視窗只依賴視窗訊息的 `WM_MOUSELEAVE`，在視窗因 hover 改變尺寸後，游標事件可能遺失，造成 hover 無法展開、展開後無法互動，或離開後不收合。
2. 右鍵沒有獨立處理 `WM_RBUTTONUP`，在透明／分層視窗的 hit-test 或預設訊息路徑不穩定時，右鍵選單可能完全不出現。
3. `refreshInFlight_` 改變後未立即要求分層畫面重繪，且畫面缺少明確的完成／失敗回饋，使用者無法判斷手動刷新是否真的完成。
4. 任務模式固定 46px 正方形，兩列剩餘量在實際 DPI 與字體下過於擁擠。
5. 英文目前仍以 Quantico 顯示；本次改用附件的 Story Script 字體。中文維持 Iansui，既有 Quantico 檔案保留以避免無關資源與既有部署路徑被破壞，但不再作為英文顯示字體。

既有用量計算、帳號處理、重新整理 API、計時器、資料讀取、玻璃背景、拖曳定位、完整／簡單／任務模式、透明度設定、唯讀重設額度庫存與繁體中文文案均保留。

## 2. 互動狀態與視窗幾何

### 2.1 懸浮球模式

完整／簡單模式仍沿用目前面板位置設定：面板是可拖曳的主要矩形，懸浮球位於面板右側並保留既有間距；不把面板移到懸浮球上，也不讓兩者重疊。

狀態規則：

- `Bubble`：視窗只包含去背 Codex 圖示；游標進入圖示即進入 `HoverExpanded`。
- `HoverExpanded`：顯示完整面板與放大的懸浮球；游標離開整個面板＋懸浮球群組後立即回到 `Bubble`。
- `PinnedExpanded`：左鍵點擊後固定顯示；移開游標不收合，再次點擊懸浮球、右上角 `×` 或使用既有關閉操作才回到 `Bubble`。

### 2.2 游標狀態來源

保留 `WM_MOUSEMOVE`／`WM_MOUSELEAVE` 作為即時反應，但新增 40ms 的輕量游標輪詢作為分層視窗的保底來源。輪詢只在完整／簡單模式且未開啟設定面板時工作：

- `Bubble` 且游標位於目前懸浮球矩形：轉為 `HoverExpanded`。
- `HoverExpanded` 且游標不在目前視窗矩形：轉為 `Bubble`。
- `PinnedExpanded` 或任務模式：不因游標改變狀態。

收合時重新計算視窗矩形；游標若仍停在懸浮球上，保留既有「點擊收合後先離開再重新 hover」的抑制規則，避免狀態在同一個游標位置來回切換。

### 2.3 右鍵

新增 `WM_RBUTTONUP` 與 `WM_NCRBUTTONUP` 的明確路徑，將訊息座標轉成螢幕座標後呼叫現有 `ShowContextMenu`。`WM_CONTEXTMENU` 仍保留以支援鍵盤／系統觸發；開啟選單前將視窗設為前景，選單位置若無有效座標則使用目前游標位置。右鍵不改變 hover／固定展開狀態。

`WM_NCHITTEST` 對目前視窗矩形統一回傳 `HTCLIENT`，不再讓分層懸浮球因透明像素或短暫尺寸轉換而把右鍵／滑鼠訊息交給背景視窗。

## 3. 任務模式尺寸與內容

- 任務模式仍是單一直徑 96px 的液態玻璃圓形元件，不使用懸浮球、不 hover 展開、不顯示大面板。
- 圓形直徑實際依 DPI scaling；固定最小寬高同步提升，避免載入設定或拖曳後被縮回 46px。
- 兩列以 `5h`、`W` 標籤和右側剩餘百分比呈現；窗口不可用時顯示 `--`，不虛構 5 小時用量。
- 兩列與底部狀態色條維持三層 liquid glass：外框、次級資訊卡、完全不透明文字／數值／控制色。
- 任務模式不因 hover 輪詢改變矩形，右鍵選單與透明度設定仍可使用。

## 4. 刷新回饋

手動或計時器啟動刷新時，先將 `refreshInFlight_` 設為真並立即重繪；完成資料回傳或失敗時立即重繪一次。完整／簡單模式的刷新圖示仍使用附件 `assets/icons/undo.png`，不加實心按鈕外框；只在刷新中以狀態色與柔和光暈強化。

刷新區域旁新增可讀狀態：

- 進行中：`Refreshing`／`重新整理中`。
- 成功：`Updated HH:MM:SS`／`已更新 HH:MM:SS`，時間取本次成功回傳時間。
- 失敗：`Update failed`／`更新失敗`，保留既有錯誤資料顯示。

倒數仍顯示下一次自動刷新時間；狀態文字與倒數不互相覆蓋。刷新按鈕 hit target 保持足夠大小，不以可見外框破壞既有無框圖示風格。

## 5. 字型與圖示

### 5.1 字型

- 新增並部署 `assets/fonts/StoryScript-Regular.ttf`，來源為使用者提供的 `Story Script` 字體檔。
- 英文與拉丁字元的 DirectWrite family 使用 `Story Script`。
- 中文與 CJK 字元維持 `Iansui`。
- 不重新命名既有 Quantico 字體資源；它們不再被英文文字 layout 選用。

### 5.2 圖示層

不引入外部網路依賴或不存在的 Flaticon API；以既有提供的 Codex／undo PNG 加上 Direct2D 向量線稿，建立一致的 icon vocabulary。圖示只作為資訊分組與控制提示，不取代高對比文字：

- 5 小時與週限額卡：時鐘／週期線稿。
- 週期速度卡：速度波形／節奏線稿。
- 重設額度卡：票券／重設循環線稿。
- 用量詳情卡：長條圖線稿。
- 刷新控制：使用附件 undo 圖示；fallback 才使用向量循環箭頭。
- 關閉控制：細線 `×`，維持透明玻璃表面。

向量圖示使用完全不透明或高對比筆畫，卡片仍保留玻璃穿透與邊緣高光，不新增實心方形按鈕。

## 6. 非本次範圍

- 不恢復官方暫時取消的 5 小時用量。
- 不修改 endpoint、認證、帳號資料流程、重設額度庫存讀取或資料解析規則。
- 不把任務模式重新變成懸浮球。
- 不把外部圖示下載、網路資源或 CDN 引入執行時依賴。

- Presentation tests 驗證 hover 狀態轉移只依目前游標是否在 bubble／window，任務模式不使用浮動球，任務尺寸為直徑 96px。
- Source contract 驗證 Story Script 資源、英文字型 family、右鍵訊息處理、hover poll timer、刷新狀態與既有 reset consume 禁止規則。
- Debug／Release CMake build、CTest、直接 VS18 build 全部通過。

### UI smoke

- 完整模式：圖示 hover 展開、移開後收合、點擊後固定、右鍵選單可出現、刷新中與成功狀態可辨識。
- 簡單模式：同上，且不與懸浮球重疊。
- 任務模式：直徑 96px 圓形、上下半部可讀 5 hr／Week 剩餘量、hover 前後尺寸一致、沒有懸浮球。
- 變更前後保留使用者 settings.ini 的位置、模式與透明度設定。
