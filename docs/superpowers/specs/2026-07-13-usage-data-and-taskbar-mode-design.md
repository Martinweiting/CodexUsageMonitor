# 用量資料容錯與任務模式小方格規格

日期：2026-07-13  
範圍：CodexUsageMonitor 原生 Win32／Direct2D 小工具

## 1. 目標

本次修正聚焦兩個目前可重現的問題：

1. 官方暫時移除 5 小時用量窗口後，週限額仍可取得，但目前解析器會因缺少任一窗口而把整份回應判定為失敗。
2. 任務模式目前仍使用「資訊面板 + 懸浮球」群組；需求改為只顯示靠近工作列的小方格，不進行 hover 展開，也不建立懸浮球。

既有 API 請求、帳號處理、重新整理、計時器、拖曳／定位、玻璃渲染、完整／簡單模式與重設額度庫存的唯讀行為均保留。

## 2. 用量資料契約

### 2.1 窗口獨立可用

`UsageWindow` 新增 `available` 狀態，代表該窗口至少成功取得可用的 `used_percent` 數值。每個窗口分開解析，並以 `limit_window_seconds` 判斷實際用途，不把 JSON 節點名稱當成固定業務語意：

- 約 5 小時的窗口（目前官方值為 `18000` 秒）對應 5 小時窗口。
- 約 7 天的窗口（目前官方值為 `604800` 秒）對應週限額窗口。
- 若兩個窗口的秒數都無法辨識，才回退到既有的 `primary_window`／`secondary_window` 順序。
- `primary_window` 目前可能承載 7 天窗口；它不能直接當成 5 小時窗口。
- 兩者都不存在或都沒有有效 `used_percent` 時，整份用量回應才算失敗。
- 只有其中一個窗口存在時，整份回應仍算成功；缺少的窗口維持不可用。

### 2.2 剩餘量計算

對可用窗口使用同一條明確公式：

`remaining_percent = clamp(round(100 - used_percent), 0, 100)`

`used_percent` 仍以官方回應為來源，不透過重設時間、方案名稱或其他欄位反推。時間欄位是可選的：

- `limit_window_seconds`、`reset_after_seconds`、`reset_at` 存在且型別正確時才填入。
- 缺少時間欄位不影響剩餘百分比顯示。
- `reset_at` 為空或無效時，時間文字顯示 `--`。

### 2.3 5 小時窗口暫時不可用

不修改 API 請求、不虛構 5 小時數值，也不以週限額代替。5 小時窗口缺少時：

- 完整模式、簡單模式與任務小方格顯示 `--`／「目前無法取得」。
- 不視為 0% 或 100%。
- 不參與「已用罄」、「吃緊」、最低剩餘量與玻璃背景色計算。
- 若週限額仍有效，週限額、週期進度與重新整理狀態照常顯示。

## 3. 顯示狀態修正

- 每個百分比、進度條、重設倒數與預計用完時間均以該窗口的 `available` 判斷，不再只看整體 `snapshot.success`。
- 週期速度只在週限額可用且窗口長度有效時建立；5 小時缺少不會讓週期卡片失效。
- 狀態色只依可用窗口判定：任何可用窗口為 0% 才是耗盡；可用窗口低於門檻或週期超前才是吃緊。
- 背景玻璃染色取所有可用窗口中的最低剩餘量；沒有任何可用窗口時使用中性載入色。
- 本地倒數只遞減可用窗口的倒數，避免不可用窗口被填入假資料。

## 4. 任務模式小方格

### 4.1 幾何

- 任務模式固定為單一 `46 x 46` DPI-scaled 的液態玻璃小方格，靠近目前螢幕工作列邊緣停靠。
- 任務模式不使用右側 Codex 懸浮球、不使用面板與 14px 群組間距。
- 不因 hover、點擊或離開而改變視窗大小。
- 工作列停靠、置頂、右鍵選單、重新整理與透明度設定保留。

### 4.2 內容與互動

- 小方格內以兩列緊湊資訊顯示 `5h` 與 `W`；可用時顯示剩餘百分比，不可用時顯示 `--`。
- 使用與其他模式相同的玻璃層級、邊緣高光、陰影、透明度與狀態色。
- 滑鼠移動不觸發 `HoverExpanded`，左鍵不切換固定展開狀態；右鍵仍開啟現有選單。
- 從任務模式開啟「透明度」時，設定畫面使用單一設定玻璃面板，不補回懸浮球。

## 5. 驗證要求

### 自動驗證

- JSON fixture：5 小時缺少／週限額有效時仍解析成功，週限額剩餘量正確。
- JSON fixture：兩窗口皆有效時，剩餘量公式與時間欄位正確。
- JSON fixture：只有無效 `used_percent` 時解析失敗。
- Presentation tests：不可用窗口不會造成耗盡或錯誤最低剩餘量；週限額有效時週期摘要仍正常。
- Source contract：任務模式固定 46px、沒有 taskbar bubble／hover 展開路徑，且既有重設額度消耗 API 仍不存在。
- Debug／Release CMake build、CTest、直接 VS18 build 均通過。

### UI 煙霧驗證

- 完整模式與簡單模式仍可由懸浮球 hover 展開，面板保持在懸浮球左側且不重疊。
- 任務模式只有 46px 小方格，hover 前後視窗矩形一致，畫面中沒有 Codex 懸浮球。
- 任務模式的右鍵「透明度」仍能開啟設定並拖曳滑桿，關閉後回到小方格。
- 顯示缺少 5 小時資料時為 `--`，週限額顯示實際剩餘百分比。

## 6. 非本次範圍

- 不恢復官方暫時取消的 5 小時用量。
- 不修改帳號認證、API endpoint、額度重設庫存讀取或其他既有資料流程。
- 不重新設計完整／簡單模式的既有玻璃質感與懸浮球互動，只修正它們對部分用量資料的顯示判斷。
